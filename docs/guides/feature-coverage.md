## Known Documentation Boundaries

Not every built-in adapter or MCP wrapper gets a separate runnable example. Instead, examples cover each config pattern:

- API token + allowlist: GitHub, Slack, Notion, Tavily, Firecrawl, Apify, ElevenLabs.
- Read-only local boundary: filesystem, database.
- Injected host client: database, storage, Redis-like clients.
- Hosted or stdio MCP wrapper: GitHub MCP and the MCP lifecycle guide.
- Media generation or analysis: multimodal inputs and image/audio adapters.

When adding a new feature, add either a runnable example or update this matrix with the nearest existing example and the reason a separate example is unnecessary.
