# Agent Creation Config

`Agent.create()` defines the default identity and provider behavior for an agent. Use it for settings that should apply across many runs: model choice, credentials, retry policy, default tools, default tool policy, logger, and default generation controls.

## Fields

| Field              | Required          | Where it plugs in                        | How it changes the result                                                                                                                                          | Use case                                                                                           |
| ------------------ | ----------------- | ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------- |
| `model`            | Yes               | Provider registry and protocol selection | Selects model capabilities, provider request format, streaming/tool support, and pricing metadata. Stronger models usually improve quality at higher cost/latency. | [Basic chat](../../examples.md#basic-chat), [Provider routing](../../examples.md#provider-routing) |
| `apiKey`           | Provider-specific | Provider SDK/client auth                 | Allows cloud provider calls. Omit for local providers unless your gateway requires auth.                                                                           | [Basic chat](../../examples.md#basic-chat)                                                         |
| `name`             | No                | Logs, traces, teams, audit output        | Makes runs easier to identify when multiple agents participate. Does not alter model behavior directly.                                                            | [Agent team](../../examples.md#agent-team)                                                         |
| `system`           | No                | Default system instruction               | Changes tone, role, constraints, and response behavior across runs. Use for durable product behavior.                                                              | [Skill composition](../../examples.md#skill-composition)                                           |
| `temperature`      | No                | Provider generation params               | Lower values are more deterministic; higher values produce more varied output.                                                                                     | [Basic chat](../../examples.md#basic-chat)                                                         |
| `maxTokens`        | No                | Provider generation params               | Caps output length and cost. Too low can truncate useful answers.                                                                                                  | [Cost budgeting](../../examples.md#cost-budgeting)                                                 |
| `timeout`          | No                | Provider call wrapper                    | Fails slow calls instead of hanging. Useful for user-facing latency targets.                                                                                       | [Provider routing](../../examples.md#provider-routing)                                             |
| `topP`             | No                | Provider sampling params                 | Alternative sampling control. Usually leave unset unless you are intentionally tuning generation.                                                                  | [Basic chat](../../examples.md#basic-chat)                                                         |
| `frequencyPenalty` | No                | Provider generation params               | Reduces repeated phrasing where supported.                                                                                                                         | [Writing and skill flows](../../examples.md#skill-composition)                                     |
| `presencePenalty`  | No                | Provider generation params               | Encourages introducing new topics where supported.                                                                                                                 | [Research agent](../../examples.md#research-agent)                                                 |
| `stopSequences`    | No                | Provider generation params               | Stops output at known markers. Useful for protocol-style prompts.                                                                                                  | [Prompt assembly](../../examples.md#prompt-assembly)                                               |
| `responseFormat`   | No                | Provider native response mode            | `json_object` requests native JSON mode where supported. Still validate with `responseSchema` when correctness matters.                                            | [Structured output](../../examples.md#structured-output)                                           |
| `tools`            | No                | Default tool collection                  | Makes tools available to every run. Prefer per-run tools when access should be narrow.                                                                             | [Custom adapter](../../examples.md#custom-adapter)                                                 |
| `toolPolicy`       | No                | Default tool execution policy            | Applies approvals, guardrails, redaction, timeouts, retries, and audit behavior to tools.                                                                          | [Guardrails and approvals](../../examples.md#guardrails-and-approvals)                             |
| `retry`            | No                | Provider retry wrapper                   | Retries transient provider failures. Higher attempts improve resilience but can increase latency and cost.                                                         | [Provider routing](../../examples.md#provider-routing)                                             |
| `logger`           | No                | Runtime logging                          | Bridges AgentCraft logs into your application logger.                                                                                                              | [Observability tracing](../../examples.md#observability-tracing)                                   |

## Provider Connection Variants

| Provider family         | Required config                    | Optional config                  | Difference in behavior                                                                                                           |
| ----------------------- | ---------------------------------- | -------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| OpenAI                  | `apiKey`                           | `baseUrl`, `organizationId`      | Default OpenAI endpoint unless `baseUrl` points at a compatible gateway.                                                         |
| OpenAI-compatible cloud | `apiKey`                           | `baseUrl`                        | Uses OpenAI-shaped requests for providers such as Groq, Mistral, Together, Fireworks, Perplexity, OpenRouter, xAI, and Cerebras. |
| Azure OpenAI            | `apiKey`, `endpoint`, `deployment` | `apiVersion`                     | `deployment` controls the Azure deployment called, which may differ from the underlying model name.                              |
| Local providers         | None                               | `baseUrl`                        | Uses local OpenAI-compatible endpoints such as Ollama, LM Studio, vLLM, or LocalAI.                                              |
| Bedrock                 | `region`                           | `accessKeyId`, `secretAccessKey` | Prefer the AWS default credential chain or short-lived roles over static keys.                                                   |
| Vertex AI               | `project`                          | `location`                       | Prefer workload identity or application default credentials.                                                                     |

## Benefits

Use agent-level config when a setting is stable and reusable. Omit it or move it to run params when a setting is task-specific, user-specific, permission-specific, or cost-sensitive.
