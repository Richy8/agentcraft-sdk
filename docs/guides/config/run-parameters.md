# Run Parameters

Run parameters are passed to `agent.run()`, `agent.stream()`, and related execution methods. They define what should happen now, for this specific task.

## Fields

| Field                                                          | Required | Where it plugs in               | How it changes the result                                                                                                                                                                                | Use case                                                               |
| -------------------------------------------------------------- | -------- | ------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------- |
| `prompt`                                                       | Usually  | Prompt resolver                 | Inline task text. Can use `&#123;&#123;var&#125;&#125;` when `vars` are provided and `&#123;&#123;config.path&#125;&#125;` when `assembly.config` is provided. Includes are rejected for inline prompts. | [Basic chat](../../examples.md#basic-chat)                             |
| `promptFile`                                                   | No       | Prompt assembler file mode      | Loads an explicit prompt entry file. The entry file may compose other prompt files with `&#123;&#123;include path&#125;&#125;`.                                                                          | [Prompt assembly](../../examples.md#prompt-assembly)                   |
| `vars`                                                         | No       | Variable injector               | Replaces `&#123;&#123;name&#125;&#125;` placeholders with task data. Arrays become markdown lists.                                                                                                       | [Prompt assembly](../../examples.md#prompt-assembly)                   |
| `assembly`                                                     | No       | Prompt assembler options        | Controls config injection, strictness, minification, partial depth, root safety, and custom loading.                                                                                                     | [Prompt assembly](../../examples.md#prompt-assembly)                   |
| `system`                                                       | No       | Run prompt construction         | Adds task-specific system guidance. Use when behavior should vary per request.                                                                                                                           | [Skill composition](../../examples.md#skill-composition)               |
| `temperature`, `maxTokens`, `topP`, penalties, `stopSequences` | No       | Provider request                | Overrides agent defaults for one run.                                                                                                                                                                    | [Basic chat](../../examples.md#basic-chat)                             |
| `tools`                                                        | No       | Tool collection                 | Adds tools only for this run. Safer than global tools for narrow tasks.                                                                                                                                  | [Custom adapter](../../examples.md#custom-adapter)                     |
| `toolPolicy`                                                   | No       | Tool execution wrapper          | Merges with agent policy and controls this run's approvals, guardrails, and limits.                                                                                                                      | [Guardrails and approvals](../../examples.md#guardrails-and-approvals) |
| `images`, `audio`, `video`, `files`                            | No       | Provider multimodal request     | Adds non-text inputs when the selected model supports them.                                                                                                                                              | [Skill composition](../../examples.md#skill-composition)               |
| `responseFormat`                                               | No       | Provider native format          | Requests text or JSON mode. Pair JSON mode with validation.                                                                                                                                              | [Structured output](../../examples.md#structured-output)               |
| `responseSchema`                                               | No       | Structured output parser        | Validates JSON and returns `structuredResponse`.                                                                                                                                                         | [Structured output](../../examples.md#structured-output)               |
| `structuredOutput`                                             | No       | Structured output controller    | Adds repair retries or tool-based schema fallback.                                                                                                                                                       | [Structured output](../../examples.md#structured-output)               |
| `signal`                                                       | No       | Provider cancellation           | Cancels long or abandoned requests.                                                                                                                                                                      | [Provider routing](../../examples.md#provider-routing)                 |
| `trace`                                                        | No       | Observability runtime           | Emits human-readable trace entries and machine-readable spans.                                                                                                                                           | [Observability tracing](../../examples.md#observability-tracing)       |
| `budget`                                                       | No       | Budget preflight/runtime checks | Blocks runs when token, cost, run, or tool limits are exceeded.                                                                                                                                          | [Cost budgeting](../../examples.md#cost-budgeting)                     |
| `replay`                                                       | No       | Replay mode                     | Uses deterministic responses for demos/tests without live provider calls.                                                                                                                                | [Replay mode](../../examples.md#replay-mode)                           |

## Choosing Agent Defaults vs Run Params

Use `Agent.create()` for stable defaults. Use run params for anything tied to a single user, task, permission boundary, cost budget, output contract, or tool list.

`promptDir` is deprecated and rejected at runtime. Use `promptFile` with an explicit entry file and `&#123;&#123;include path&#125;&#125;` directives instead.
